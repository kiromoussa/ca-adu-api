# ADU Atlas API - developer portal

Next.js 14 (App Router) + TypeScript + Tailwind developer portal for the ADU
Atlas API. Deployed to Vercel; the API itself runs on Render (see
`../docs/adr/0001-architecture.md`).

## Pages

- `/` - landing page: the core promise, what a feasibility response
  contains, the no-LLM-on-request-path / provenance / state-baseline
  framing, a coverage teaser, and the disclaimer.
- `/coverage` - jurisdiction coverage, generated from `../config/jurisdictions.yaml`.
  Never claims coverage ahead of verified ingestion: status badges read
  directly from the config file (production / ingesting / planned).
- `/docs` - OpenAPI 3.1 reference rendered with `swagger-ui-react`. Tries the
  live spec at `${NEXT_PUBLIC_API_BASE_URL}/openapi.json` first; if that is
  unreachable (for example in local development before the API is deployed)
  it falls back to the bundled snapshot at `public/openapi.yaml`.
- `/pricing` - BASIC / PRO / ULTRA / MEGA plan cards generated from
  `../config/plans.yaml`, with a "Get API key on RapidAPI" call to action per
  plan.
- `/changelog` - calls `GET {NEXT_PUBLIC_API_BASE_URL}/v1/changelog` at
  request time (`dynamic = "force-dynamic"`, no caching). Shows a plain
  empty/error state rather than fabricated entries if the API is not yet
  reachable.

## Configuration is data-driven, not hardcoded

Coverage claims and pricing figures are never hardcoded in this app. They are
read at build/request time from:

- `../config/jurisdictions.yaml` (coverage status, supported project types,
  official code URLs)
- `../config/plans.yaml` (plan tiers, quotas, rate limits, billing terms)
- `../config/sources.yaml` (source registry names/links used on `/coverage`)

`src/lib/server-config.ts` loads these with `js-yaml`, probing a short list
of candidate roots so it works whether the process cwd is `portal/`
(standard Vercel "Root Directory = portal" setup) or the monorepo root.

### Vercel monorepo setting

If the Vercel project's Root Directory is set to `portal`, enable the
project setting **"Include source files outside of the Root Directory in
the Build Step"** so `../config/*.yaml` remain readable at build time.
`next.config.js` also sets `outputFileTracingRoot` to the repo root for the
same reason.

## Environment variables

See `.env.example`:

- `NEXT_PUBLIC_API_BASE_URL` - base URL of the FastAPI service (no trailing
  slash). Used by `/docs` (live spec) and `/changelog` (live entries). Safe
  to leave unset locally; both pages degrade gracefully.
- `NEXT_PUBLIC_RAPIDAPI_URL` - the RapidAPI listing URL used by every "Get
  API key on RapidAPI" button.

## Local development

```bash
npm install
cp .env.example .env.local   # edit as needed
npm run dev
```

This repository intentionally does not vendor `node_modules` or a lockfile
generated by CI; install before running locally or deploying.

## Style conventions

- No emojis, no em dashes (hyphens only) anywhere in copy or code.
- `cursor: pointer` is set explicitly on every button and link (Tailwind
  `cursor-pointer` class plus a global `a, button { cursor: pointer }` rule
  in `globals.css`).
- The exact disclaimer string (`src/lib/constants.ts`, `DISCLAIMER_TEXT`)
  matches the product spec verbatim and must not be edited without updating
  the spec.

```
    page.tsx              landing page

    pricing/page.tsx        /pricing

  components/
    Nav.tsx, Footer.tsx, Disclaimer.tsx, Badge.tsx, RapidApiCta.tsx

    types.ts               shared config/API types
    server-config.ts        YAML config loader (server-only)
    constants.ts             disclaimer text, env accessors, labels
public/
  openapi.yaml             bundled OpenAPI fallback snapshot (see /docs)
```
